## 🏗️ Архитектура
Бот построен на **n8n** — low-code платформе для автоматизации. Весь backend — это один воркфлоу без единой строчки серверного кода.

```
Пользователь
    │
    ▼
[Telegram] ──webhook──▶ [n8n Workflow]
                              │
                    ┌─────────┴─────────┐
                    │                   │
               /start?              обычное сообщение
                    │                   │
             приветствие          [GigaChat OAuth]
                    │                   │
             [Telegram API]       [Google Docs]
                                  загрузка промта
                                        │
                                  [Get context]
                                  сборка истории диалога
                                        │
                                  [GigaChat API]
                                  генерация ответа
                                        │
                                  [Telegram API]
                                  отправка ответа
                                        │
                                  [Save response]
                                  сохранение в контекст
```

| Нода | Роль |
|---|---|
| **Webhook** | Принимает все входящие апдейты от Telegram |
| **Set** | Хранит плейсхолдеры для токенов (bot_token, api_key) |
| **If** | Разделяет `/start` и обычные сообщения |
| **Send greetings** | Отправляет приветствие на двух языках (RU + KZ) |
| **uuid** | Генерирует уникальный RqUID для OAuth-запроса GigaChat |
| **Set token** | Получает Bearer-токен через GigaChat OAuth 2.0 |
| **Get a prompt** | Подтягивает актуальный системный промт из Google Docs |
| **Get context** | Собирает историю диалога пользователя (до 10 последних сообщений) и формирует массив messages для модели |
| **Create request** | Формирует запрос к GigaChat Chat Completions API |
| **Send response** | Отправляет ответ модели пользователю в Telegram |
| **Save response** | Сохраняет ответ ассистента в историю диалога пользователя |
| **Register webhook** | Утилита для регистрации webhook (запускается один раз) |

## 💬 Контекст диалога

Бот поддерживает **многоходовый диалог** — помнит, о чём шла речь ранее:

- История сообщений хранится в `staticData` (глобальное состояние n8n-воркфлоу) — отдельно для каждого пользователя по его Telegram ID
- Каждое сообщение пользователя добавляется в историю, после получения ответа — ответ ассистента тоже сохраняется
- Окно контекста ограничено **10 последними сообщениями** (5 пар «вопрос-ответ»), что предотвращает переполнение и контролирует расход токенов
- При формировании запроса к модели собирается массив: `[system prompt] + [история диалога]`

Пример:
```
Пользователь: Где находятся ваши магазины?
Бот: У нас 4 шоурума в Алматы...

Пользователь: А какой из них самый большой?
Бот: (понимает, что речь идёт о магазинах из предыдущего сообщения)
```

## 🧠 Подход к системному промту
Информация о компании собрана вручную с [centr-krasok.kz](https://centr-krasok.kz): главная страница, контакты, каталог, страницы для партнёров (дизайнерам, строителям), описание брендов и услуг.

Данные структурированы в системный промт по разделам:

- Роль и личность бота
- Защита от prompt injection
- О компании (юрлицо, позиционирование)
- Адреса и контакты (все шоу-румы, телефоны, email, соцсети)
- Полный каталог товаров
- Услуги (колеровка, доставка, мастер-классы и др.)
- Условия для партнёров (дизайнеры, строители)
- Все бренды (40+)
- Вакансии
- Правила ответов (антигаллюцинационные ограничения)

**Ключевые решения:**

- Промт хранится в **Google Docs** — его можно обновить в любой момент без изменения воркфлоу и деплоя
- Модель явно инструктирована **не выдумывать цены и артикулы** — при отсутствии данных направляет пользователя к менеджеру
- Встроена **защита от prompt injection**: бот игнорирует попытки сменить роль через фразы «забудь всё», «новая роль», «DAN» и т.п.

## 🔐 Безопасность
Все чувствительные данные **вынесены из кода** и заменены плейсхолдерами:

```
bot_token  →  <your_bot_token>
api_key    →  <your_api_key>
```

В продакшене заполняются через переменные окружения n8n или секреты платформы.

## ⚙️ Установка и настройка

### 1. Клонировать репозиторий
```bash
git clone https://github.com/khamid95/centr-krasok-test
```

### 2. Импортировать воркфлоу в n8n
В интерфейсе n8n: **Workflows → Import from file** → выбрать `AI bot.json`

### 3. Заполнить учётные данные
В ноде **Set** подставить:
- `bot_token` — токен Telegram-бота (получить у [@BotFather](https://t.me/BotFather))
- `api` — GigaChat API ключ в формате Base64 (из [личного кабинета Сбера](https://developers.sber.ru/studio))

### 4. Подключить Google Docs
- В ноде **Get a prompt** указать ссылку на документ с системным промтом
- Настроить OAuth 2.0 credentials для Google Docs в n8n

### 5. Зарегистрировать webhook
Запустить ноду **Register webhook** один раз — она привяжет Telegram к вашему n8n-инстансу.

### 6. Активировать воркфлоу
Переключить тоггл **Active** в n8n.
